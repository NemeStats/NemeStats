# UI MODULE

**Generated:** 2026-05-20
**Scope:** ASP.NET MVC 5 web application, Web API v2, SignalR, frontend assets.

## OVERVIEW
Web application layer. MVC controllers delegate to BusinessLogic retrievers. Web API is versioned via URL (`/api/v2/...`). SignalR hubs live in the separate `NemeStats.Hubs` project.

## STRUCTURE
```
UI/
├── Controllers/         # MVC controllers (9 total; AccountController ~645 lines)
├── Areas/Api/           # Web API v2 controllers + models
├── Views/               # Razor views (match controller names)
├── Transformations/     # ViewModel builders / AutoMapper profiles
├── App_Start/           # RouteConfig, BundleConfig, WebApiConfig
├── Scripts/             # JavaScript (Namespace.js pattern)
├── stylesheets/         # Sass → CSS (Grunt build)
├── sitemaps/            # XML sitemaps (generated by WebJob)
└── Mappers/             # AutoMapper configuration
```

## WHERE TO LOOK
| Task | Location | Notes |
|------|----------|-------|
| Add MVC controller | `Controllers/` | Inherit `BaseController`; use T4MVC links |
| Add API controller | `Areas/Api/Controllers/` | Versioned under `v2/` |
| Add view | `Views/<Controller>/` | Name action after controller method |
| Add JS for a view | `Scripts/` | Namespace matches file path; see `Namespace.js` |
| Add bundle | `App_Start/BundleConfig.cs` | Use `Links.Bundles.*` for T4MVC safety |
| Add route | `App_Start/RouteConfig.cs` | Prefer attribute routes |
| Add AutoMapper map | `Mappers/` | Called from `AutomapperConfiguration` in Global.asax |

## CONVENTIONS
- **Controllers are thin** — delegate to BusinessLogic; no domain logic here.
- **T4MVC**: Use `MVC.Player.Details(...)` instead of magic strings for URLs.
- **JS Namespace pattern**: `Namespace("UI.Views.Player");` at top of file.
- **Bundles**: Script bundles defined in `BundleConfig.cs`; use T4MVC `Links` when possible.
- **Sass**: Compiled via Grunt; `gruntfile.js` at project root.

## ANTI-PATTERNS
- **Do not edit `*.generated.cs` files** — T4MVC outputs.
- **Do not put domain logic in controllers** — move to `BusinessLogic/Logic/`.
- **Do not reference `System.Web` from BusinessLogic** — keep layers clean.
- **Do not check in `PrivateAppSettings.config`** — contains secrets.

## UNIQUE STYLES
- **Grunt + Sass**: Legacy frontend build; `npm install` + `grunt` to compile.
- **Vue.js**: Used sparingly for reactive UI components (see bundle config).
- **SignalR**: Hubs in separate `NemeStats.Hubs` assembly; referenced via script bundle.
